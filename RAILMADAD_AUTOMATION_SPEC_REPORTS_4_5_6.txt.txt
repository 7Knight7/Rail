SECTION 3 — SHARED RECEIVED-SORTING RULE
============================================================

This rule applies to Reports 4, 5 and 6.

1. Click Submit before attempting to sort.

2. Wait for the correct result table to load.

3. Identify the received-count column in the correct table.

Depending on the report, the header may be:

- Received
- Feedback Received

4. Use the header belonging to the current result table.

5. Do not select a similarly named column from another table on the same page.

6. Perform two separate click actions:

   a. click the received-count header once

   b. wait for the table to update

   c. click the same header a second time

   d. wait for the table to become stable

7. Do not use Playwright dblclick().

8. Read the first several non-empty values.

9. Parse the values as numbers.

10. Verify:

    value[0] >= value[1] >= value[2] ...

11. If descending order is not verified:

    - retry the complete two-click sequence once

12. If the second attempt fails:

    - save screenshot
    - save page HTML
    - raise SortError
    - mark the current report or type as failed

13. Do not continue with unsorted data.
